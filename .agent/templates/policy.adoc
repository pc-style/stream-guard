= Governance Policy

== Mandatory Gates
. `compile_gate`: All code must compile/build successfully
. `check_gate`: All AsciiDoc cross-references must resolve, no cycles, valid edges
. `heal_scan_changed_only`: Only scan files changed since last commit
. `security_gate`: Never commit rebuildable `.aden/store`, caches, generated contracts, or secret artifacts. Durable `.aden/` configuration/hooks may be committed when intentional. Run `aden ci-check` before every commit.

== Confidence-Based Escalation
|===
|Confidence |Action |Cost

|>= 0.95 |Auto-apply |0 tokens
|>= 0.85 |Requires one-line ACK |~500 tokens
|>= 0.70 |Requires inline review |~2000 tokens
|< 0.70 |Full semantic review |~4000 tokens
|===

== Token Budgeting
- Context assembly: `--budget 8192`
- Review pipeline: `--budget 2048`
- Per-session limit: 10000 tokens total

== Session Lock Protocol
. Every agent appends to `.agent/session.adoc` before starting
. Acquire an exclusive lock on `.agent/session.adoc.lock` (for example `flock .agent/session.adoc.lock`) before reading or writing
. Append atomically under lock: read current content, merge the new row, write to a temp file in the same directory, then rename/replace into place
. If lock acquisition fails after a short backoff, re-read and retry; resolve conflicting rows by timestamp ordering
