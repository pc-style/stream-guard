=== Action Item Categories

Every action item must be one of:

* **Prevent** — Make the failure impossible next time
* **Detect** — Catch the failure earlier, when it is cheaper
* **Mitigate** — Reduce blast radius when it happens again
* **Document** — Update runbooks, playbooks, or decisions

== Prevent Recurrence

. What process change makes this failure harder?
. What code change catches this earlier?
. What monitoring gives us warning?
