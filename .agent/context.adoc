== Hard Constraints

. **Never use npm**; use Bun for any JavaScript tooling.
. **Keep the repo buildable with Apple Command Line Tools only** unless a user explicitly chooses full Xcode.
. **Never commit generated build artifacts** such as `.aden/`, `contracts/`, `.build/`, or `node_modules/`.
. **Do not treat Aden as verification**; after changes, still run repo checks such as `make build` and `make test`.
. **Avoid live OCR/UI tests by default** unless screen-recording permission and a visible capture window are intentionally available.

== Language and Style Defaults

Aden is supplemental. The authoritative source of truth is always the source
code, docs, and tests in this repository.

== Repo Workflow Notes

. Start with `AGENTS.md`, `README.md`, `docs/ARCHITECTURE.md`, and `Makefile`.
. Prefer Aden for navigation over raw grep/find/cat when you need to understand unfamiliar code.
. Keep local scratch/log folders such as `.codex-maintenance/` and `.stream-guard-test-logs/` ignored by both Git and Aden.
. Commit only intentional repo guidance/config; do not commit `.aden/store`, caches, or generated contracts.
